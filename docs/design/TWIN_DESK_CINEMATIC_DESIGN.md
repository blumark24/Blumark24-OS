# مكتبي الذكي — حزمة التصميم السينمائي (My Twin Desk — Cinematic Design Package)

> **الحالة: بانتظار الاعتماد — لا يبدأ أي تنفيذ برمجي قبل الموافقة على هذه الحزمة.**
>
> **المعاينة البصرية الكاملة (الإطارات + التوكنز + المكونات):**
> https://claude.ai/code/artifact/22fba9bc-2498-4ec7-9b9a-d22266bd48cb

## 1. الهدف

إعادة تصميم `مكتبي الذكي` كوضع غامر بملء الشاشة (Immersive Mode) يجعل الموظف يشعر أنه **داخل مكتب رقمي ذكي فاخر**، لا أمام لوحة مهام اعتيادية — بجودة مؤسسية سعودية، عربية أولاً (RTL)، وبمنطق توأم رقمي صادق: كل عنصر بصري يحمل معنى تشغيليًا حقيقيًا.

## 2. الاتجاهات البصرية الثلاثة والقرار

| الاتجاه | الخلاصة | القرار |
|---|---|---|
| أ · المكتب التنفيذي الفاخر | دفء كهرماني وعمق مادي، لكنه وحده «ديكور» | يُستبقى منه: الإضاءة الكهرمانية السينمائية |
| ب · مركز القيادة الذكي | رادار/مؤقتات/مؤشرات — كل بكسل له معنى تشغيلي | **المعتمد — هيكل التجربة** |
| ج · الخريطة التكتيكية | أقوى تجسيد للتوأم الرقمي، لكنه يغيّب «وش أسوي الآن؟» | يُستبقى منه: طبقة الخريطة المصغرة + شاشة المدير |

**التركيب النهائي:** هيكل (ب) + إضاءة (أ) + طبقة خرائط (ج).

## 3. الشاشات

- **سطح المكتب 1440×1024:** شريط قيادة علوي → ثلاثة أعمدة: «مهامي اليوم + المهمة القادمة» يمينًا (372px)، المشهد السينمائي في المنتصف، «رادار التنبيهات + مؤشر ضغط العمل» يسارًا (296px) → شريط سفلي: مسار المهمة الإداري + مؤقت التصعيد + خريطة المكتب الافتراضي → شريط إبحار زجاجي حول جرم «B».
- **الجوال 390×844:** تركيبة مستقلة أولويتها «وش أسوي الآن؟» — بطل سينمائي مضغوط (132px)، بطاقة المهمة القادمة + «ابدأ العمل» فورًا، ثم رادار/حالة/خريطة مصغرة، وشريط إبحار سفلي.
- **رادار الفريق (المدير) 1440×960:** رادار أقسام كبير + بطاقات حمل الأقسام + قائمة «يحتاج انتباهك» بأفعال سياقية (اعتماد · إعادة توزيع · طلب توضيح · تصعيد للإدارة) + شريط حالة الفريق. خلف صلاحية إدارية من RBAC القائم.

## 6. حقيقي مقابل تمثيلي في MVP (لا بيانات وهمية في الواجهة)

| العنصر | المصدر |
|---|---|
| مهام اليوم/الإحصاءات/المهمة القادمة وأزرارها | حقيقي — `useTasks()` و`update()` والتوست القائم |
| اسم الموظف وقسمه | حقيقي — `useAuth()` + الهيكل الإداري |
| مؤشر ضغط العمل | مشتق حقيقي من المهام النشطة/المتأخرة (صيغة معلنة) |
| رادار التنبيهات | مشتق من المهام؛ «طلبات المساعدة» فقط عند توفر مصدرها |
| مؤقت التصعيد | مشروط بوجود قاعدة تصعيد/SLA — وإلا يُخفى بصدق (لا عدّ وهمي) |
| خريطة المكتب/أحمال الأقسام | تمثيلي بوسم «معاينة» حتى يتوفر مصدر حمل الأقسام، أو يُخفى للمنشآت بلا أقسام |
| المشهد السينمائي/الهولوغرام | بصري خالص (CSS/SVG) بمعنى: حالة المقعد تعكس حالة الموظف الحقيقية |

## 7. قواعد التنفيذ

- `src/app/tasks/my-desk/page.tsx` تُعاد بناؤها كوضع غامر بلا الشريط الجانبي العام، مع بقاء `PageGuard permission="manage_tasks"`.
- شاشة المدير: `src/app/tasks/team-radar/page.tsx` خلف صلاحية إدارية.
- المشهد CSS/SVG خالص: لا صور نقطية، لا WebGL، لا مكتبات جديدة (`framer-motion` القائمة تكفي).
- الحركة: `transform/opacity` فقط + احترام `prefers-reduced-motion`؛ لا مؤقتات JS إلا للعدادات.
- متجاوب: ≥1440 الشبكة الكاملة؛ 1024–1439 تُطوى الجوانب؛ <768 تركيبة الجوال المستقلة.
- RTL: `dir="rtl"` + خصائص منطقية في المكونات الجديدة (مع مواضع فيزيائية داخل المشهد حيث يلزم).
- حالات فارغة صادقة: «لا توجد مهام نشطة اليوم» / «لا تنبيهات حاليًا» بدل أرقام مزيفة.
- الأداء لألف عميل: نفس جلب البيانات الحالي، صفر استعلامات جديدة، صفر أصول ثقيلة.

## 8. مراحل التنفيذ بعد الاعتماد

1. توكنز `twin.*` في Tailwind (إضافة تراكمية).
2. مكتبة مكونات `twin-desk` مع القصص/الحالات الفارغة.
3. شاشة سطح المكتب الغامرة (بيانات حقيقية من الأول).
4. تركيبة الجوال.
5. رادار الفريق (خلف الصلاحية).
6. بوابة الجودة: lint + build + مراجعة RTL/الهوية قبل أي دمج.
